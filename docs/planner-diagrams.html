② Auto planner — step by step

Pick a date, pick an appointment, tell the system what you committed to Amazon, and it builds the truck for you. Five steps from start to dispatch. 

flowchart TD
  Start(["User opens
New Shipment"]) --> Mode["Picks Auto Planner"]

  Mode --> Step1["STEP 1 - Pick a date"]
  Step1 --> API1["System loads dates that
have appointments waiting"]

  Step1 --> Step2["STEP 2 - Pick an appointment"]
  Step2 --> API2["System loads appointments
for that date with their orders"]
  Step2 --> API2b["Also loads other orders
at the same warehouse
(extras you can add in)"]

  Step2 --> PickPOs["User picks one or
more appointments,
unchecks unwanted orders,
or adds extras"]

  PickPOs --> Step3["STEP 3 - Set things up"]
  Step3 --> Commit["Popup asks: how many
units and cartons did
you commit to Amazon?
(saved for next time)"]
  Commit --> CommitOpts{"User decides:
Strict ?
Skip ?
Use only units or cartons ?"}
  CommitOpts --> Cfg["Pick truck size
+ Choose how to split load
(Premium / Commodity / Other)
+ Top up empty space ?"]

  Cfg --> Generate(["Click Generate Plan"])
  Generate --> Pool["System gathers eligible items
(matching warehouse, in stock,
not already on another plan)"]
  Pool --> Score["Each item scored by urgency
(low stock = high priority)"]
  Score --> Pack["Brain fits as many items as
possible into the truck
(full cartons only)"]
  Pack --> Fillers{"Top up turned on ?"}
  Fillers -- yes --> Same["Adds more items from
the same appointment"]
  Fillers -- yes --> DOH["Adds urgent items from
other orders at the
same warehouse"]
  Same --> Caps["Removes anything over
Amazon's commitment cap"]
  DOH --> Caps
  Fillers -- no --> Caps
  Caps --> Suggest["If truck is mostly empty,
suggest a smaller one"]
  Suggest --> Resp["Sends finished plan back:
what loaded, what didnt,
why short, totals"]

  Resp --> Step4["STEP 4 - Review the plan"]
  Step4 --> Edit["User can edit shipped qty,
remove rows,
switch truck size,
refresh plan"]
  Edit --> Save(["Click Save Draft"])
  Save --> StrictGate{"Strict mode on ?
Plan matches commitment ?"}
  StrictGate -- yes / matches --> Persist["Plan is saved to
the database"]
  StrictGate -- mismatch --> Block["Error: numbers dont match -
fix qty or turn strict off"]
  Block --> Edit

  Persist --> Step5["STEP 5 - Send for approval"]
  Step5 --> Approver["Plan is now pending approval"]
  Approver --> Approve["Manager approves"]
  Approve --> Dispatch["Team marks the truck dispatched"]
  Dispatch --> Done(["Shipment dispatched"])

③ Manual planner — step by step

No appointment needed. Pick the exact orders and items you want to ship, give your commitments per order, and the same planner brain works out the truck. 

flowchart TD
  Start(["User opens
New Shipment"]) --> Mode["Picks Manual"]

  Mode --> Load["System loads every
pending purchase order
across all warehouses"]
  Load --> Lookup["Checks what is already
on other plans so we
dont double count"]
  Lookup --> Net["Shows the real remaining
quantity for each item"]
  Net --> Filters["User filters by order,
item name, warehouse,
category, or searches"]

  Filters --> Pick["User clicks orders or
individual items to add
them to the basket"]
  Pick --> FC{"All picks from the
same warehouse ?"}
  FC -- no --> Warn["Error banner:
pick from one warehouse only"]
  Warn --> Pick
  FC -- yes --> Cfg["User picks truck size"]

  Cfg --> Generate(["Click Generate Plan"])
  Generate --> CommitGate{"Any picked order
without a commitment ?"}
  CommitGate -- yes --> Modal["Popup asks for units and
cartons for each order"]
  Modal --> SaveModal["User fills in or hits Skip"]
  SaveModal --> Generate
  CommitGate -- no --> RunBrain["System sends items to
the planner brain"]

  RunBrain --> Score["Each item scored by urgency"]
  Score --> Sort["Sorted by score, expiry,
quantity"]
  Sort --> Pack["Brain fits items into
the truck (full cartons only)"]
  Pack --> Caps["Removes anything over the
commitment cap (per order)"]
  Caps --> Resp["Sends finished plan back"]

  Resp --> Nav["Takes user to Review page"]
  Nav --> Review["User can edit quantity,
remove rows,
change item or code"]
  Review --> Persist(["Click Save Draft"])
  Persist --> Saved["Plan is saved to
the database"]

  Saved --> Submit["Send for approval"]
  Submit --> Approve["Manager approves"]
  Approve --> Dispatch["Team marks the truck dispatched"]
  Dispatch --> Done(["Shipment dispatched"])
